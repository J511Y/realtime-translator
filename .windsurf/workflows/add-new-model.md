# 모델 추가/교체하기

1. 모델 분류 확인
   - translation(음성/텍스트 번역)인지
   - ocr(텍스트 추출)인지
   - vision(이미지 이해 + 응답 생성)인지

2. 환경변수 추가
   - 필요한 API 키/엔드포인트를 `.env.example`에 추가한다.
   - 키 이름은 공급사별로 명확히 네이밍한다.

3. 서버 어댑터 추가
   - `lib/` 하위에 공급사별 클라이언트를 만든다.
   - 공통 인터페이스(입력/출력 타입)를 정의해 UI와 분리한다.

4. UI 모델 선택 반영
   - 모델 선택 UI에 옵션을 노출한다.
   - 기본값/폴백 전략을 정의한다.

5. 테스트
   - 최소 1개 샘플 입력으로 통합 테스트를 수행한다.
   - 실패 시 사용자 메시지가 기술 정보를 노출하지 않도록 한다.
